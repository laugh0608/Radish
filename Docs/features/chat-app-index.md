## 概述

本组文档聚焦聊天室在 `radish.client` 中作为独立应用的前端架构与交互实现，按“架构/实时/模块/里程碑”拆分，避免单文档持续膨胀。

当前额外提供纯 Web `/messages` 入口，用于普通浏览器登录态会话 / 消息复访。`/messages` 复用现有 `ChatApp`、聊天 API 与 `ChatHub`，承接通知里的 `channelId/messageId` 定位和成员公开主页来源返回；它不是完整聊天平台迁移，不新增私聊、搜索、Reaction、置顶、阅读回执或移动系统通知。

后台数据模型与 API 细节请优先参考：
- [聊天室系统设计](./chat-system.md)
- [聊天室系统 - 前端架构与组件设计](./chat-frontend.md)
- [纯 Web 私域复访入口设计说明](/frontend/private-web-revisit)

## 阶段命名约定

本组文档并行使用两套编号，含义不同：

| 编号 | 出现位置 | 含义 |
|------|----------|------|
| Phase 1 / 2 / 3 | `chat-system.md` | 后端系统能力的全量规划（按模块整体交付） |
| P0 / P1 / P2 | `chat-app-roadmap.md` | 前端落地节奏（按最小可演示里程碑拆分） |

对应关系：P0 = Phase 1 核心子集；P1 = Phase 1 剩余能力；P2 = Phase 2；Phase 3 暂不在 roadmap 中展开。

## 交付边界

- 默认覆盖 Chat App（WebOS 窗口应用）实现；`/messages` 只作为纯 Web 复访壳复用当前 Chat App，不改变后端聊天协议。
- 不覆盖 Console 管理页细节。
- 默认复用现有 `@radish/ui` 与公共基础设施（认证、上传、日志、通知）。
- 不新增重复协议；优先复用 `chat-system.md` 已冻结的数据契约。
