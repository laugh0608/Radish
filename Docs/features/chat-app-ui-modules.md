## 总体布局

标准三栏：
- 左栏：频道导航（分类、频道、未读）
- 中栏：消息主区（置顶预览条 [Phase 2] · 消息历史 · 输入框）
- 右栏：成员列表（在线/离线，可折叠）

响应式策略：
- 宽度 < 850px：隐藏右栏
- 宽度 < 650px：左栏折叠为抽屉或图标模式

## 模块拆分

1. `ChannelSidebar`
- 展示频道分类树和频道项。
- 负责本地过滤、折叠状态持久化、未读标记显示。
- 不直接请求消息接口。

2. `MessageArea`
- 负责消息虚拟滚动、历史分页、自动滚底策略。
- 承载消息分组渲染和“新消息提示条”。

3. `MessageInput`
- 负责输入、发送、快捷键、图片上传草稿、@mention 下拉。
- 复用 `StickerPicker` 插入 `sticker://`。
- 图片选择后会先调用现有上传接口，上传成功再进入输入区的待发送附件区，等待用户点击发送。
- 支持直接粘贴剪贴板图片，并将结果写回当前频道草稿。
- `@mention` 下拉会跟随输入光标定位，确保插入反馈与候选列表位置一致。

4. `MemberList`
- 展示在线成员与状态。
- 支持收起/展开，移动端默认隐藏。

5. `PinnedMessageBar`（Phase 2）
- 展示在消息区顶部（消息历史上方，频道标题下方）。
- 显示置顶消息的发送者名称与内容截断（50 字符）。
- 点击跳转至消息历史列表中的原始消息位置。
- Moderator/Owner 可点击"×"取消置顶；普通成员只读。
- 无置顶消息时完全收起，不占位。

## 核心组件职责

`MessageBubble`：
- 渲染文本、贴纸、图片、撤回占位。
- 提供 hover 操作（引用、撤回、Reaction）。
- 当前基础布局为“头像在侧边、元信息在上方、消息内容在下方”；自己消息采用左右镜像排布。
- Phase 2：hover 菜单增加"📌 置顶"选项（仅 Moderator/Owner 可见）。
- Phase 2：气泡右下角展示阅读回执区，最多 3 个已读在线成员头像（18×18），超出显示 `+N`。

`MessageGroup`：
- 将同一用户短时间连续消息合并显示。
- 减少头像与昵称重复渲染，提升密度。

`ReplyPreview`：
- 在输入区顶部展示引用目标。
- 支持一键取消引用。

`MentionDropdown`：
- 输入 `@` 后触发用户搜索。
- 键盘上下选择 + Enter 确认。

`ReadReceipt`（Phase 2）：
- 位于消息气泡右下角。
- 展示 `lastReadMessageId ≥ messageId` 的在线成员头像，至多 3 个，按已读时间倒序排列。
- 离线成员不展示，避免历史消息出现大量僵尸头像。

## 关键交互规范

1. 自动滚动
- 用户位于底部时，新消息自动吸附到底部。
- 用户浏览历史时，不强制跳转，仅显示“新消息”提示。

2. 发送行为
- `Enter` 发送，`Shift+Enter` 换行。
- 存在待发送图片时，发送按钮会把“当前文字 + 待发图片”一起发出；若没有文字，也支持发送纯图片。
- 图片上传进行中时，发送按钮保持禁用，避免半成品附件被误发。
- 发送失败时保留失败消息卡片用于重试 / 撤销，不再依赖保留输入框内容。
- 图片发送成功后不额外弹出成功提示，只在失败时提示，避免高频发送被重复打断。

3. 撤回展示
- 撤回消息保留时间与发送者区域。
- 内容替换为统一占位文案“[已撤回]”。

4. 图片消息
- 缩略图最大 300x300，点击进入灯箱。
- 选择图片后先上传，上传成功后在输入区显示待发送附件预览，不会立即写入消息列表。
- 直接粘贴图片时沿用同一条上传与草稿链路，不绕过待发送附件区。
- 待发送附件支持发送前移除；真正发送后仍沿用现有图片消息结构与灯箱预览。
- 上传中显示进度，失败后可重新选择。

5. 草稿持久化（P1）
- 草稿统一存入 `localStorage` 的 `radish.chat.drafts.v1`，按 `userId + channelId` 维护多频道草稿。
- 包含输入文本、`replyTarget`（引用状态）与 `pendingImage`（待发送图片）。
- 切换频道时自动保存当前草稿，恢复目标频道的草稿内容。
- 若图片上传完成时用户已切到其他频道，上传结果会回写到原频道草稿，避免误发。
- 发送成功后清除对应草稿。

6. 消息置顶（Phase 2）
- 置顶条位于消息历史与频道标题之间；格式：`📌 [发送者]: [内容截断 50 字]`。
- 右侧有"×"按钮（仅 Moderator/Owner 可见）；点击条目跳转到原始消息（虚拟滚动 scrollToIndex）。
- 置顶/取消后 Hub 广播 `MessagePinned`，置顶条实时更新或收起。

7. 阅读回执（Phase 2）
- 仅展示在线成员且 `lastReadMessageId ≥ messageId` 的已读头像。
- 仅对最新 5 条消息渲染回执区，更早的消息不渲染（减少 DOM 负担）。
- 自己发的消息不展示自己的已读头像。

## 主题与样式约束

- 复用 WebOS 主题变量，不单独定义体系外颜色。
- 保持与 Forum 的字号层级一致，降低认知切换成本。
- 动效仅用于必要反馈（面板展开、新消息提示、输入中状态）。

---

## 可访问性基线

- 关键操作按钮提供 `aria-label`。
- 焦点顺序遵循“频道 -> 消息 -> 输入区”。
- 色弱场景不依赖单一颜色表达未读状态，辅以数字/图标。
