# 2026 年 4 月开发日志

> 社区体验收口与 `Phase 2-2` 启动月，重点对齐论坛输入体验、通知回跳链路与公开内容壳层首批落地。

## 本月摘要

- 论坛正文图片坏图链路已收口：`MarkdownRenderer` 当前已统一放行 `attachment://` / `sticker://`，数据库重建后重新发帖也不再因为渲染层过滤导致正文图片损坏。
- 发帖正文、评论区与聊天室输入当前都已接通 `@提及` 联想；评论区与聊天室也已补齐粘贴图片上传，社区输入体验完成一轮统一化收口。
- 编辑帖子默认重新回到正文编辑态，并修复了编辑器只剩单行窄输入区的样式回归。
- 通知中心已为非聊天类通知补齐触发者头像兜底，服务端头像字段缺失时会按公开资料回填，减少默认占位头像。
- OIDC 登录回调完成后会先预热当前用户资料，再返回桌面壳层，缩短头像 / 等级信息迟到刷新的空窗体感。
- 聊天室图片发送成功提示已移除，仅保留失败提示；相关提及弹窗定位也已完成一轮细节收口。
- WebOS 窗口打开逻辑已补齐首次居中与按应用记忆位置 / 尺寸，避免固定偏移开窗带来的违和感；论坛“正文下轻回应墙”已先沉淀为专题设计草案，暂不直接并入当前阶段主线文档。
- `Phase 2-2` 当前已正式切入实现：forum 公开移动入口首批已通过 `/forum` 与 `/forum/post/:postId` 落地，公开帖子详情首批只承载正文、轻回应墙展示与评论阅读；同时 forum 前端构建切块与 `MarkdownRenderer` warning 也已同步收口。
- forum 公开移动入口当前已完成第一轮阅读体验收口：公开列表状态已支持 URL 回写与“详情 -> 返回列表”上下文保留，详情页已补齐只读阅读摘要；公开列表卡片也已新增 `publicCompact` 变体，手机上不再继续沿用桌面右侧信息栏密度。
- forum 公开移动入口后半段问题当前也已补洞完成：问答回答编辑区高度与背景断层已修正，较宽窄屏 / 平板下的双栏卡片摘要节奏已微调，公开详情评论区的子评论分页阅读也已接通。
- forum 维护池内的评论与抽奖链路当前也已完成一轮收口：聊天室成员栏收起态已去重为单一入口，论坛两层评论已补齐“回复锚点”表达，帖子卡片神评预览已切回主数据优先，抽奖规则也已统一为“截止时间至少晚于发帖 1 小时、作者满 1 小时后可提前开奖、到点自动开奖”。
- forum 当前主线直接相关的外部 ID 字符串化残留本月也已完成收口：公开阅读、通知跳转、浏览回跳、深链 / 窗口参数、神评批量补查与抽奖通知 payload 当前都已不再依赖前端数值型外部 ID。
- 公开 docs 阅读首批当前已落地并完成一轮人工验收收口：`/docs` 与 `/docs/:slug` 已可直接进入公开文档目录与正文阅读，返回目录会保持原目录滚动位置，正文内链与旧 `__documents__` 链接也会继续落入公开 docs 壳层。
- docs 公开阅读本月又完成一轮文档收口：公开阅读、公开搜索、公开分享与 `/discover` 来源返回当前已拆成独立专题清单，并回收到统一回归索引与总基线入口。
- 个人公开页首批当前已接入公开内容壳层：`/u/:id` 可直接查看公开资料、公开统计、公开帖子与公开评论；公开 forum 中的作者点击也已开始回落到该入口，不再只能经桌面 `profile` 窗口进入。
- 公开个人页本月也继续补了一轮首屏只读说明增强：`/u/:id` 当前已新增“公开主页阅读说明”，用来解释基础资料、公开帖子 / 评论阅读与工作台边界，避免把公开资料页误读成完整个人中心。
- 公开榜单首批当前也已接入公开内容壳层：`/leaderboard` 与 `/leaderboard/:type` 已可直接查看榜单切换、分页和公开排行内容；登录用户会额外看到“我的排名”，用户榜单项也已开始跳到 `/u/:id`，而商品榜单首批仍保持只读展示，不直接打通商城购买链路。
- 公开榜单本月又补了一条更合适的增量方向：默认经验榜单页 `/leaderboard`（兼容 `/leaderboard/experience` 收口）当前已增加“经验体系公开展示”说明区，用来解释排行依据、等级含义与只读边界，但不会把“我的经验明细 / 经验交易记录”这类强登录个人页面直接搬进公开壳层。
- 公开榜单本月也继续把只读说明扩到非经验榜单：用户榜单当前会补“公开比较 + 公开个人页跳转”的轻量阅读提示，商品榜单当前会补“只读展示 + 不带购买 / 订单流程”的边界提示。
- 公开商城浏览首批当前也已接入公开内容壳层：`/shop`、`/shop/products` 与 `/shop/product/:productId` 已可直接查看首页、商品列表与商品详情；商品详情接口已回收到匿名读取口径，而购买、订单、背包和举报等登录后工作台链路继续留在桌面壳层。
- forum 公开分类首批当前也已接入公开内容壳层：`/forum/category/:categoryId` 已可直接进入分类阅读页，首批会补齐分类标题、简介、帖子数与分类下帖子列表阅读；旧 `/forum?category=...` 也会继续兼容并收口到新路径。
- forum 公开标签首批当前也已接入公开内容壳层：`/forum/tag/:tagSlug` 已可直接进入标签阅读页，首批会补齐标签名称、简介、帖子数与标签下帖子列表阅读；帖子卡片和帖子详情里的标签点击当前也已统一收口到该直达路径。
- forum 公开标签当前已继续收口到 canonical ASCII slug：旧中文标签名链接进入后，会在公开壳层内回写到新的 `/forum/tag/:tagSlug`；不存在的 `tagSlug` 也会继续停留在只读状态页，而不是静默回退到全部帖子列表。
- forum 结构化帖子类型直达当前也已补齐：`/forum/question`、`/forum/poll` 与 `/forum/lottery` 已接入公开内容壳层，帖子卡片和帖子详情里的“问答 / 投票 / 抽奖”徽标会统一回落到对应类型列表。
- forum 公开标签与后台读取链路当日也补了一轮 SQLite 稳定性收口：标签 canonical slug 唯一性检查已避开 SQLite 下的非法 SQL 生成；仓储层针对本地默认 SQLite 的通用读查询已增加同步回退，启动期 `ShopJob / PostLotteryJob` 不再因为 `reader is closed` 这类异步 reader 问题反复报错。
- repo quality 门禁回归本月也已完成一次收口：仓库卫生与 changed-only 校验重新对齐，`PR -> master` 的最小质量闸门恢复到可合并状态。
- 公开社区分发页首批当前已完成 `/discover` 落地，并补齐 forum / docs / leaderboard / shop / `u/:id` 顶部统一“社区发现”回流入口，公开壳层开始形成稳定总入口。
- 公开详情返回链路本月又完成一轮收口：从 `/discover`、`/u/:id` 与其他公开来源进入 forum / docs 详情后，返回动作当前会优先回到原来源页，而不是退回专题默认列表；公开个人页也已开始按同一口径保留来源返回。
- 公开内容壳层本月又完成一轮稳定性批量验收：`/docs/search`、`/leaderboard`、`/shop/products` 与 `/discover` 的 replace 路由同步、同路径 no-op、防抖回写、越界页码 / 无效参数规范化与来源返回当前都已收口，公开入口不再频繁出现循环更新和参数抖动。
- 公开内容壳层当前已形成共享头部品牌口径：forum / docs / discover / leaderboard / shop / `u/:id` 在窄屏下统一使用同一套品牌字、图标与按钮色 token；discover 摘要和 leaderboard 卡片的信息编排也已改回更适合横向扫读的节奏。
- 公开社区分发页本月又继续补了一轮高价值只读阅读增强：`/discover` 首屏当前已把 forum / docs / leaderboard / shop 四张摘要卡收口为“先预览本页区块，再决定是否直接进入公开页”的双动作节奏，同时继续同步 ready / loading / error / empty 状态，不再退化成纯数字入口。
- `/discover` 本月还接通了 `section` 级来源回记：摘要卡预览、区块内“查看全部”和内容跳转会记住最近一次公开分发区块，后续从 forum / docs / leaderboard / shop / `u/:id` 顶部点击“社区发现”回流时，会优先回到原区块而不是页首。
- 附件公开资源误清理本月已补根因治理：当前新增附件引用检查器，并在下载链路自动恢复仍被业务引用但状态异常的附件记录；公开商城、榜单与社区发现页的图片 404 不再继续靠前端兜底掩盖。

## 当前结论

- `Phase 2-1` 的社区体验收口项当前已达到“可转维护”的状态。
- `Phase 2-2` 当前已开始以公开内容壳层推进 forum 首批移动入口，但仍保持增量迁移口径，不调整 `/` 与 `/desktop` 的根入口关系。
- `Phase 2-2` 当前已从“入口首批已落地”推进到“公开阅读体验第一轮已收口”，下一步应优先按人工验收清单做窄屏复核，而不是回头继续把桌面 forum 交互塞回公开壳层。
- `Phase 2-2` 当前又从“公开阅读体验第一轮已收口”继续推进到“公开入口稳定性批量验收已完成一轮”，下一步更适合转入剩余入口与真实联调观察，而不是继续在同类页面上叠更多局部同步补丁。
- `Phase 2-2` 当前这一批公开内容壳层入口已基本完成稳定性收口，默认转入稳定维护；后续若继续推进，更适合沿现有公开入口补高价值只读说明与阅读增强，而不是回头把强登录桌面能力直接搬进公开壳层。
- forum 公开壳层当前范围仍保持在“只读阅读”边界内；后续默认优先继续做阅读体验补洞与验收复核，不把完整桌面互动直接搬进公开壳层。
- 公开内容壳层当前已从“按入口逐个落地”继续推进到“总入口与来源返回链路收口”，后续更适合继续围绕外链分享、来源保持与人工验收补洞推进，而不是回头扩桌面工作台语义。
- 公开社区分发页当前已从“入口首批已落地”继续推进到“摘要卡预览本页区块 + `section` 来源回记”的导览增强阶段；后续更适合优先做真实联调复核与剩余回流细节观察，而不是回头扩桌面工作台能力。
- forum 标签独立公开直达当前已进入已落地事实；`Post/GetList` 已补齐 `tagSlug` 原生筛选，公开标签页当前收口为 `/forum/tag/:tagSlug`，并继续保持只读阅读边界。
- forum 维护池内的评论治理、神评稳定性与抽奖规则本轮也已完成代码和文档收口；后续优先进入真实联调与观察，而不是继续扩张新范围。
- forum 主线相关的外部 ID 字符串化补洞当前已达到可收口状态，后续默认转入稳定维护，不再继续把这类 forum 联调尾项当作主线扩张。
- 公开 docs 阅读首批、个人公开页首批与公开榜单首批当前都已开始进入真实落地；`Phase 2-2` 后续应继续围绕公开内容壳层做移动阅读体验复核与边界收口，而不是回头给桌面工作台追加窄屏补丁。
- 附件公开资源协议当前不仅完成“去 URL 真值化”口径，也补上了“仍被业务引用的附件不应被清理误删”这一层运行时稳定性要求；后续若再改商品、榜单、发现页等图片读取链路，应继续围绕业务引用真值治理，而不是追加前端坏图兜底。
