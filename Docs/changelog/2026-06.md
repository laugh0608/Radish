## 当前结论

- `P3-8-D` 仍是当前主线，但不继续扩展完整移动商城、完整通知中心、完整创作器或 WebOS 新功能。
- 受控写入可以继续推进成熟 API 支撑的单一用户动作，但每次必须覆盖登录回流、错误态、返回状态和定向测试。
- `ID Phase A` 的下一步不是完整 `PublicId` 全量迁移或数据库主键迁移，而是先补 Console / Flutter / Web 外部 LongId 字符串安全检查，防止 `Number(...Id)`、`voUserId: number`、`uuid: number` 等回潮。
