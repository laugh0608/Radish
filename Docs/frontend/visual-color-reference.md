> 本页用于整理 Radish 首版 `dev` 的主题配色母板，作为后续 **国风视觉基线、主题切换与 UI 改造** 的统一取值参考。

## 1. 来源说明

当前主题母板采用你提供的“人民币纸币色系”作为起点，但并不直接把某一张纸币的颜色整张铺满界面，而是将其提炼为：

1. 基础中性色
2. 品牌主色
3. 模块辅助色
4. 功能状态色
5. 纹样 / 边饰点缀色

这样做的目标是：

- 保留东方审美中的温润、含蓄和层次感；
- 避免直给的大红大绿和“国潮营销页”质感；
- 让社区 UI 更适合长期使用，而不是一次性的风格展示。

## 3. 推荐角色映射

### 3.1 基础中性色

用于页面背景、面板底、分割线和阅读区底色。

推荐来源：

- `#f3e7e9`
- `#e5e1c8`
- `#cfe0d0`
- `#d4e1f1`
- `#e7d5be`

使用原则：

- 大面积背景优先选浅暖灰、米白、纸色，不直接使用高饱和色。
- 深色中性只用于标题、正文、边框和信息层级，不用于大面积涂底。

### 3.2 品牌主色

用于品牌强调、主按钮、激活态和当前焦点。

推荐来源：

- 主品牌色：`#b24057`
- 主品牌悬停：`#cd5076`
- 主品牌柔化底：`#cd94b2`

使用原则：

- 100 元色系不用于整页满铺，只用于视觉焦点。
- 主品牌色在同一屏内不要与过多高饱和辅助色同时争抢视觉中心。

### 3.3 模块辅助色

用于不同应用或页面气质的分层，不用于替代品牌主色。

推荐来源：

- 玉青 / 水碧：`#437371` `#4f9c83` `#73bea0`
- 墨蓝 / 青黛：`#1f2d38` `#414d61` `#435c74`
- 赭石 / 肉桂：`#824d3e` `#886349` `#b5826d`
- 雅紫 / 灰藕：`#604771` `#846992` `#d5bfc2`

建议分工：

- 论坛 / 社区主应用：主品牌色 + 墨蓝系
- 聊天室 / 实时消息：玉青系 + 墨蓝系
- 文档 / 阅读界面：纸色系 + 赭石 / 墨蓝系
- 商城 / 物品与权益：暖赭系 + 玉青系

### 3.4 功能状态色

状态色不能完全照搬通用红黄绿，而应基于母板做中国风语境下的柔化处理。

推荐映射：

- 成功：`#4f9c83`
- 警告：`#b5826d`
- 错误：`#b24057`
- 信息：`#435c74`
- 禁用：`#6e736d` 或对应模块低饱和浅色

## 5. 使用约束

### 5.1 要做

- 用浅底、深字、低饱和层次作为整体基础。
- 用主品牌色控制焦点（如主按钮、右键菜单的伪印章微标），而不是控制整页背景。
- 用辅助色区分模块气质，但保持整体系统一致性。
- 让纹样色比正文和边框更轻，始终退后一级（例如使用 `0.05~0.15` 透明度的 SVG `mask-image` 映射 `var(--theme-pattern-edge)`）。
- 善用排版：大标题引入衬线字体（宋体 / `var(--theme-font-serif)`），正文维持无衬线黑体，增大 `line-height`，通过实体的间距制造“留白感”。

### 5.2 不要做

- 不要把 100 元色系大面积铺成整页主背景。
- 不要在同一个组件上同时使用多个强饱和主题色。
- 不要把状态色做成互联网通用霓虹色。
- 不要为了“国风”强行加金色描边、厚重纹样或满屏装饰。
