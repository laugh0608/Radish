# 可恢复错误与诊断复制

本文定义 `radish.client` 正式 Web 页面遇到可恢复错误时的用户反馈和支持诊断边界。

## 适用场景

可恢复错误指用户可以继续操作、稍后重试或向支持人员反馈的问题，包括：

- 发帖、评论、聊天消息发送失败。
- 评论附件、图片或文档上传失败。
- 页面级数据加载失败，例如公开 Docs、资产流水、订单列表、订单详情和背包。
- 通知目标缺失、历史目标已删除或当前无法解析。
- Workbench 只同步到部分活动数据。

这些场景不能只显示 toast，也不能静默退化为空列表或伪造跳转。

## 交互契约

- **保留输入与本地状态**：发帖、评论和聊天失败后，草稿或本地失败消息必须保留。
- **给出恢复动作**：页面级错误优先提供重试；必要时提供返回目录、返回订单列表或进入通知 / 消息等稳定入口。
- **允许复制上下文**：用户无法自行恢复时，提供“复制诊断 / 复制上下文”动作。
- **避免误导**：通知目标缺失时不渲染 `#` 链接，不把用户带到错误的默认页。
- **说明恢复边界**：SignalR 断连、活动同步不完整等状态应说明当前列表仍可阅读，恢复后会继续追平。

## 诊断内容

页面级诊断统一使用：

```typescript
import { copyRecoveryDiagnostics } from '@/utils/recoveryDiagnostics';

await copyRecoveryDiagnostics({
  module: 'shop.private',
  stage: 'order-detail',
  error,
  target: { orderId },
  context: { routeKind: 'order-detail' },
});
```

诊断文本只包含：

- `module` 与 `stage`
- 当前浏览器路径
- 错误摘要
- 目标对象标识，例如 `orderId`、`slug`、`channelId`、`messageId`
- 有限上下文，例如分页、筛选、route kind、是否存在目标

诊断内容不得包含支付口令、登录密码、token、完整请求体、隐私正文、附件内容、后端堆栈或未公开文档正文。

## 当前落地点

- 论坛发帖失败：提示草稿仍保留，可稍后重试。
- 评论附件上传失败：提示评论草稿仍在编辑器中。
- 聊天发送失败：本地失败消息可重试、撤销或复制诊断。
- 通知目标缺失：保留通知上下文并允许复制支持信息。
- Workbench 活动同步失败：生成可恢复队列项，指向通知或消息入口。
- 公开 Docs、资产、订单和背包加载失败：提供重试与复制诊断。

该能力用于用户反馈和人工排障，不替代前端统一 logger、后端审计日志或 Console 治理证据链。
