# Console 表格布局说明

> 入口页：[Console 样式与 Token 使用说明](/frontend/console-style-guide)
>
> 最后更新：2026-06-30

本文说明 `radish.console` 表格、表格操作区、弹窗 / 抽屉内表格和分页布局的长期实现口径。它补充 [DataTable 组件](/frontend/data-table) 的通用用法，专门约束 Console 后台页面的容器、响应式和样式分层。

## 适用范围

- 高频表格 CRUD 页，例如用户、订单、商品、标签、分类、应用、贴纸分组和分组表情。
- 治理工作台内的证据表格、经验流水、治理日志和动作日志。
- Dashboard、用户详情、系统设置历史、文档版本弹窗、贴纸批量上传弹窗等内嵌表格。
- 需要固定操作列、批量操作、分页、工具条或中宽 PC / 移动窄屏可读性的页面。

## 容器规则

- 主表格页优先使用 `admin-table-layout`、`admin-table-panel`、`admin-table-toolbar` 和 `admin-table-aside`。
- 嵌入 Dashboard、详情页、Modal、Drawer 或批量上传流程的表格，使用 `admin-table-scroll-region` 或页面等价滚动容器。
- 表格父容器必须设置可收缩边界，例如 `min-width: 0` 和 `max-width: 100%`；不要靠扩大页面 `min-width` 解决列拥挤。
- 表格仍可使用 Ant Table / `@radish/ui` 的 `scroll={{ x: ... }}`，但横向滚动应该被限制在当前表格区域内。

## 操作列

- 固定右侧操作列必须给稳定 `width`。
- 操作按钮组使用 `Space wrap` 或等价 `flex-wrap`；按钮多时允许换行，不压缩按钮文案。
- 操作列只放当前行的直接动作；跨对象批量动作放工具条，详情跳转或排障上下文放摘要栏或详情入口。
- 删除、停用、恢复、重试、回滚等高风险动作继续使用确认弹窗和权限判断，不因为布局迁移改变业务动作。

## 分页与工具条

- 分页器所在容器需要允许 `flex-wrap`，移动窄屏下可换行。
- 筛选工具条使用可换行布局，输入框和下拉框保留合理最小宽度。
- 分页、工具条和摘要栏不能遮挡表格，也不能把主操作按钮挤出可见区域。
- 禁用分页的短表格也需要检查父容器滚动，例如 Dashboard 最近订单和弹窗内预览表。

## 样式规则

- 列内正负金额、弱文本、状态色、备注换行和预格式化文本使用 CSS class + `--console-*` token。
- 不在 `render` 函数中写 inline 色值、宽度、margin 或 display。
- 深层表单、上传预览、隐藏输入、弱提示和弹窗 footer 动作区优先复用 `adminForm.css`。
- 路由状态、无权限和懒加载状态优先复用 `routerComponents.css`，不要在路由组件中写临时样式。

## 不变契约

表格布局治理只处理视觉承载和可用性，不改变：

- API 与 HTTP 客户端。
- 权限键与按钮级权限。
- 路由、URL 状态和来源返回。
- 分页参数、筛选参数和排序语义。
- 表单字段、保存动作和提交载荷。
