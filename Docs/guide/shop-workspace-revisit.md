# WebOS 商城回访链路

> 入口页：[商城系统设计方案](/guide/shop-system)  
> 最后更新：2026-05-22

本文说明登录后 WebOS 工作台里的商城回访链路。公开商城详情仍只承载浏览、分享和回流，不承载购买、订单、背包或个人资产动作。

## 1. 适用边界

- 适用：`radish.client` 的商城窗口、通知中心打开商城、商城内部购买 / 订单 / 背包来源回访。
- 不适用：公开商城 `/shop`、`/shop/products`、`/shop/product/:productId` 的只读浏览壳层。
- 不扩展：购物车、退款、实物物流、公开购买、完整运营平台或公开增长观察。

## 2. 入口参数

商城窗口当前支持两类直达参数：

| 参数 | 目标 | 典型来源 |
|------|------|----------|
| `productId` | 商品详情 | 订单详情回商品、背包权益来源回商品、背包消耗品相关商品、桌面应用内商品回访 |
| `orderId` | 订单详情 | 购买成功、订单通知、订单列表、背包来源回订单 |

参数只负责打开目标视图，不改变公开路由的 canonical 口径，也不把 long ID 暴露成公开增长主线。

## 3. 用户链路

### 3.1 购买成功

1. 用户在 WebOS 商城商品详情页确认购买。
2. 后端创建订单、扣款、扣库存并发放权益。
3. 前端收到 `orderId` 后进入订单详情。
4. 订单详情展示订单状态、商品快照、支付结果与回商品入口。

若后端没有返回有效 `orderId`，前端只保留成功提示和背包刷新，不伪造订单详情入口。

### 3.2 订单通知

1. 通知中心识别商城订单通知中的订单 ID。
2. 用户点击通知后打开或复用商城窗口。
3. 商城窗口进入对应订单详情。

通知只承接已存在的订单上下文，不在通知中心内实现订单治理、支付或售后流程。

### 3.3 订单详情回商品

订单详情页保留商品回访动作，用于从历史订单重新查看商品信息。该动作进入 WebOS 商城商品详情，不跳公开商城详情页。

### 3.4 背包来源回访

权益来源来自 `UserBenefitVo`：

| 字段 | 含义 |
|------|------|
| `VoSourceOrderId` | 购买产生的来源订单 |
| `VoSourceProductId` | 购买产生的来源商品 |

前端类型使用 `voSourceOrderId` 与 `voSourceProductId`。当来源类型为购买且字段值有效时，背包权益卡片展示“查看订单”和“查看商品”动作。

消耗品来源来自 `UserInventoryVo`：

| 字段 | 含义 |
|------|------|
| `VoSourceProductId` | 当前可回跳的相关商品 |

前端类型使用 `voSourceProductId`。当字段值有效时，背包消耗品卡片展示“相关商品”动作并打开商城商品详情。

消耗品背包 `UserInventory` 当前只保留商品来源能力，不把订单来源误写成背包物品的统一契约。

## 4. 设计原则

- 用户完成购买后优先看到订单结果，而不是只看到 toast 或被动去背包查找。
- 资产来源必须可追溯，但回访入口只展示真实存在的订单 / 商品，不做无来源兜底链接。
- WebOS 私域回访与公开商城分享分离：公开详情负责浏览和传播，私域商城负责购买、订单和背包。
- 后续新增商城入口时优先复用 `productId` / `orderId` 参数，不新增平行的跳转协议。
