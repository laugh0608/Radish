# 用户承诺与公开边界

> 本文说明 Radish 正式产品面向用户公开承诺的当前边界。公共页面入口为 `/legal`，对应前端实现位于 `Frontend/radish.client/src/public/legal/`。

## 定位

Radish 当前正式产品以社区为主轴：帖子、评论、聊天、关注和圈子是主要互动路径，Docs、商城、资产、宠物、经验、通知、Workbench 和 Console 都应服务社区内容生产、关系沉淀、信任治理和持续复访。

用户承诺页的目标不是替代完整法务文本，而是把正式发布前必须让用户理解的产品边界公开化：

- 内容和互动规则
- 公开、私域和 Console 可见范围
- 账号安全和治理处理原则
- 虚拟商品、资产、经验、宠物和权益边界
- 未成年人、敏感内容和安全反馈
- 反馈诊断所需信息

## 当前公开入口

- 公共路径：`/legal`
- 公共壳层导航：公开 Web 顶部导航和移动底部导航均提供“规则”入口
- 页面标题：`用户承诺与社区边界 - Radish`
- 访问要求：不要求登录

该入口应保持可从默认公开入口 `/discover` 到达。后续若新增注册页、设置页、帮助页或错误页，也应复用 `/legal` 作为用户承诺落点，避免散落多份互相冲突的规则文本。

## 承诺维度

### 社区内容与互动公约

- 禁止违法违规、侵权、恶意广告、欺诈误导、色情低俗、人身攻击、骚扰和恶意引战。
- 发帖、评论、回答和聊天应围绕主题展开；引用外部资料时保留必要来源。
- 点赞、轻回应、关注和举报用于表达参与和维护秩序，禁止刷赞、刷评论、恶意举报和批量养号。
- 举报目标包括帖子、评论、轻回应、聊天消息和商品；治理处理应保留目标快照、举报原因和处理记录。

### 公开、私域与 Console 可见边界

- 公开主页、公开帖子、公开评论、公开榜单和公开商品可被未登录访客查看。
- 账号凭证、资产明细、订单、附件管理和私域状态不作为公开内容展示。
- 关注关系、经验、宠物、背包和资产信息只按当前页面边界展示；扩大公开范围前必须更新说明和验证口径。
- Console 仅面向授权管理员，用于用户排障、内容治理、订单 / 权益核对、权限和系统设置管理。

### 账号、安全与使用承诺

- 用户应妥善保管账号、密码、支付口令和登录会话，不共享账号，不绕过权限或滥用接口。
- 发布、编辑、购买、关注、举报、治理和资产相关动作会按业务需要记录时间、目标对象、操作者和处理结果。
- 平台可以对明显违规、攻击、刷屏、欺诈、骚扰或破坏社区秩序的账号执行禁言、封禁、撤销处理或其他必要限制。

### 虚拟商品、资产与权益边界

- 胡萝卜、经验、背包权益、宠物状态和虚拟商品没有现金价值，不提供提现、转售或线下兑换承诺。
- 当前商城以虚拟商品和权益发放为主；购买前应确认商品名称、价格、库存、可购买状态和到账路径。
- 当前正式边界不承诺完整退款 / 售后平台；因系统错误造成的重复扣减、发放异常或订单失败，应按订单、资产流水和背包记录核对处理。
- 任何资产、订单、背包或权益异常都应能关联订单号、业务目标、用户和时间点。

### 未成年人、敏感内容与安全反馈

- 未成年人应在监护人知情和指导下使用社区互动、聊天、商城和资产相关能力。
- 不得发布自伤、暴力、仇恨、色情、赌博、毒品、诈骗、隐私泄露或其他可能伤害他人的内容。
- 遇到骚扰、威胁、欺诈、侵权、隐私泄露或疑似未成年人风险时，应优先使用举报入口并保留上下文。
- 涉及人身安全、违法犯罪或紧急风险时，应优先联系现实中的可信成年人、平台管理员或当地相关机构。

### 反馈、诊断与问题恢复

- 页面异常反馈应尽量包含当前路径、操作时间、诊断编号、截图和复现步骤。
- 内容异常反馈应提供帖子、评论、聊天消息、商品、订单、用户或举报单编号。
- 购买、背包、资产、经验或宠物状态异常应提供订单、流水、背包来源或相关页面路径。
- Console 管理员处理治理问题时，应基于目标快照、举报原因、用户历史和动作日志形成判断。

## 实现边界

- 本批不新增后端 API、权限、数据库结构、注册流程或独立帮助中心。
- `/legal` 是公开说明页，不处理表单提交、申诉工单或退款申请。
- 完整隐私政策、用户协议、退款 / 售后平台、账号注销 / 数据导出和自动化治理平台仍需后续专题继续扩展。
- 页面内容如与 `Docs/guide/shop-system.md`、`Docs/guide/console-modules.md`、`Docs/guide/forum-content-write-reliability-governance.md` 或后续正式法务文本冲突，应优先更新本文和 `/legal` 页面，保持对用户可见口径一致。

## 验证口径

改动触达用户承诺页时，至少执行：

- `npm run build --workspace=radish.client`
- `npm run check:repo-hygiene:changed`
- `git diff --check`

若改动同时触达购买、资产、治理、身份或通知行为，应按对应专题补定向测试。真实 Gateway smoke 只在阶段验收、发布候选或用户明确说明前后端已启动时执行。
