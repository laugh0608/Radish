# 正式 Web 聊天工作区使用说明

本文说明登录用户如何使用正式 Web `/messages` 聊天工作区，以及搜索、消息回应、置顶和阅读回执的可见边界。系统设计和接口契约分别见 [聊天室 App 文档总览](/features/chat-app-index) 与 [聊天室系统设计](/features/chat-system)。

## 1. 入口与页面形态

- 正式产品入口为 `/messages`，要求登录。
- PC 使用会话列表、消息区和按需展开的成员 / 搜索面板。
- mobile 使用会话列表与详情两级视图；进入会话后优先保留消息区和输入空间。
- WebOS `/desktop` 中的“聊天”复用同一 `ChatApp`、API、Store 和 ChatHub，不是另一套聊天数据。
- 通知或站内链接可使用 `/messages?channelId={id}&messageId={id}` 打开目标消息；`channelId / messageId` 是字符串标识，不应手工转为 JavaScript 数字。

## 2. 会话列表与私聊状态

会话按服务端返回的关系分区展示：

- 互相关注
- 陌生人私信
- 好友群组
- 公共频道

一对一会话可能处于 `Pending / Accepted / Declined / Blocked` 等状态。页面只展示服务端允许的接受、拒绝、阻断、解除、归档或恢复操作，不根据频道名称、成员数或本地角色推断权限。

陌生人请求未接受前，发起人只能发送一条纯文本请求；拒绝或阻断后不能继续发送。归档只影响自己的列表，不删除双方消息；合法深链仍可打开有权访问的归档会话。

## 3. 发送、草稿与恢复

- `Enter` 发送，`Shift+Enter` 换行。
- 输入 `@` 可搜索并选择成员；候选列表支持方向键、`Enter / Tab` 选择和 `Escape` 关闭。
- 图片上传完成后先进入当前会话的待发送区，点击发送后才成为消息；可以继续输入文字、移除图片或发送纯图片。
- 文本、引用目标和待发送图片按账号与频道保存草稿。上传完成时即使已经切换会话，也会回写原会话草稿，不会发到当前会话。
- 发送中的本地消息使用临时状态；失败后保留在原位置，可重试、撤销或复制诊断信息。
- 消息撤回后保留占位。撤回会同步移除该消息的活跃回应和置顶，不保留旧正文作为页面兜底。

## 4. 搜索消息

搜索支持“当前会话”和“全部可见会话”两种范围，也可以设置开始 / 结束日期：

1. 打开“搜索消息”。PC 在右侧显示搜索面板，mobile 进入独立搜索视图。
2. 输入 2 至 100 个字符；在输入框按 `Enter` 或点击“搜索”。
3. 结果按同一次服务端快照分页，点击结果会打开原会话并定位、高亮目标消息。
4. `Escape` 关闭搜索；关闭或返回后恢复原触发位置。

搜索关键词不会写入 URL、持久化存储、跨标签广播或日志。搜索仅覆盖当前账号此刻有权访问的会话；权限变化会使旧 cursor 失效，页面要求重新搜索。消息已撤回、会话失效或权限变化时，不展示缓存旧正文，也不自动跳到最新消息。

## 5. 消息回应

- 可使用快捷 emoji 或“更多表情与贴纸”回应消息。
- 同一账号在一条消息上最多保留 10 种不同回应。
- 再次选择已使用的回应会取消该回应；重复请求和多标签并发最终以服务端权威状态为准，不会因重试反向切换。
- Public / Announcement 中可读消息允许回应；普通 Private 仅有效成员可回应；Direct 仅在已接受、未阻断且对端可用时允许回应。
- 消息撤回、失权或表情停用后，页面刷新为当前有效状态；加载回应失败不清空消息历史或草稿。

## 6. 消息置顶

- 消息区顶部仅在存在置顶时显示最近置顶摘要和总数；每个频道最多保留 20 条活跃置顶。
- 点击摘要或完整列表中的“定位”会打开原消息；PC 使用紧凑面板，mobile 使用 Bottom Sheet。
- Public / Announcement 由 System / Admin 或有效 Moderator / Owner 管理；普通 Private 仅频道 Moderator / Owner 管理；已接受且未阻断的 Direct 双方都可管理共享置顶。
- 页面只消费服务端返回的 `CanPinMessages`，不会因为前端显示了某个角色名称就开放按钮。
- 达到容量上限时必须先取消其他置顶；系统不会自动移除最旧记录。

## 7. 未读与阅读回执

未读和阅读回执共用服务端 `LastReadMessageId` 游标，但用户可见语义不同：

- 未读数用于当前账号判断还有多少新消息，并在多标签之间同步。
- Public / Announcement 不向发送者展示阅读身份、头像、人数或比例。
- 普通 Private 只允许发送者查看自己消息的已读人数和当前有效读者；PC 使用 Popover / 侧浮层，mobile 使用 Bottom Sheet，并按页加载。
- 已接受且未阻断的 Direct 只在对端已读到的最后一条自己消息下显示一次“已读 / Read”边界。
- Pending / Declined / Blocked Direct 不展示对外回执，避免泄露陌生请求或阻断后的阅读行为。

打开会话、加入 Hub、搜索命中或定位历史消息都不自动等于已读。页面只有在浏览器可见且聚焦、消息区位于会话尾部，并且 WebOS 窗口未最小化且位于前台时，才提交实际展示到的最高持久消息。回执不表示对方读完、理解或在某台设备收到消息，也不记录阅读时间、设备和停留时长。

## 8. 多标签、断线与状态恢复

- `/messages` 与 WebOS 可以同时打开；它们共享账号级 ChatHub 生命周期，但各自保持页面展示状态。
- Hub 事件负责提示消息、回应、置顶、未读或回执状态发生变化；页面通过 revision 或重新读取 HTTP 权威状态防止乱序覆盖。
- 网络断开时保留已加载消息、搜索结果、草稿和失败消息。恢复后重新加入当前频道并拉取权威状态。
- 搜索、回应、置顶或回执读取失败不会清空普通消息历史，也不会把错误状态写成成功。
- 登出或切换账号会清除聊天 Store 中的频道、消息、回应、置顶和回执摘要，防止跨账号显示。

## 9. 键盘、语言与无障碍

- 搜索、置顶列表、读者详情和消息操作均使用真实按钮，可通过键盘聚焦。
- 搜索输入支持 `Enter`，搜索 / 置顶 / 读者面板支持 `Escape` 关闭。
- 读者详情关闭后恢复触发按钮焦点；mobile Bottom Sheet 使用模态语义和受控焦点边界。
- 加载、结果数量、错误和目标失效通过 `aria-live`、`role=status` 或 `role=alert` 表达，不只依赖颜色和高亮。
- 固定界面文案、日期、数字和人数复数支持中文与英文；用户消息正文、名称和贴纸名称不自动翻译。

## 10. 当前边界

正式聊天工作区当前不承担：

- 匿名公开聊天室；
- 完整私有群组创建、邀请和成员管理；
- 公告覆盖率或阅读比例分析；
- 消息编辑、转发、语音、视频或端到端加密；
- 浏览器 / 移动系统通知栏推送；
- 管理员任意浏览普通 Private 或 Direct 内容。

遇到无权访问、目标已撤回、会话状态变化或 cursor 失效时，应以页面当前提示和重新读取结果为准，不通过刷新本地缓存绕过权限。
