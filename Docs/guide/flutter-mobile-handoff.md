# Flutter 移动端 handoff 与回流说明

本文说明 Flutter Android MVP 当前已经成立的原生导航、外部来源和登录回流边界。完整客户端范围仍以 `Clients/radish.flutter/README.md` 为准。

## 总体边界

Flutter 承担移动原生客户端路线，不复刻 WebOS。当前重点是公开内容阅读、原生返回、登录续接、个人复访、最小通知回流和轻回应写入。

当前不包含：

- 完整通知中心
- 系统通知栏推送
- 完整商城工作台
- 发帖、完整评论提交、点赞、投票、编辑治理
- 桌面工作台多窗口能力

## 榜单到公开主页

原生 `榜单` tab 当前承载经验榜只读入口。

- 经验榜条目可打开原生公开主页。
- Android Back 返回榜单，而不是退到默认 profile tab 或退出应用。
- 商品榜、我的经验明细、经验交易记录和管理员调整能力不进入当前 Flutter 批次。

## Forum 通知回流

已登录壳层会读取当前用户最新一条可跳 forum 的通知作为轻入口。

- 点击后复用 forum detail handoff 打开目标帖子。
- 通知携带 `commentId` 时，详情页继续尝试定位目标评论上下文。
- 从通知进入详情后，Android Back 返回打开通知前所在 tab，不强制留在 forum tab。
- 该入口只验证站内最新 forum 通知读取；不代表完整通知中心、系统通知栏推送、标记已读、删除通知或通知设置已纳入 Android MVP。

## Forum 轻回应登录回流

Forum detail 当前按“正文 -> 轻回应 -> 评论区”展示。

- 匿名态可读取最近轻回应。
- 已登录态可发布一句轻回应。
- 从轻回应区点击“登录后发布”后，浏览器 OIDC 登录成功会回到当前帖子轻回应区，并提示可继续发布。
- 发布成功只刷新轻回应墙与局部成功提示，不刷新正文或评论阅读位置。
- 轻回应删除、举报、审核台能力、完整评论提交、点赞、投票和编辑治理仍不进入当前 Flutter 批次。

## 标识与内部接口

- 外部来源优先携带 `postPublicId`。
- 旧 `postId` 只作为字符串 fallback。
- 进入详情并读取成功后，评论、轻回应和定位类内部接口使用真实 `VoId`。
- 不应把公开标识误传给需要内部 ID 的接口。

## 验证要点

- 经验榜用户条目打开公开主页后，Android Back 回榜单。
- 从非 forum tab 点击 `Forum notification` 打开详情，返回后回到原 tab。
- 通知携带 `commentId` 时，应落到评论上下文或给出可理解的定位失败提示。
- 匿名态从轻回应区发起登录后，应回到当前轻回应区并可继续发布。
