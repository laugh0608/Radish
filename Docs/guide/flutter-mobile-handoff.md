# Flutter 移动端 handoff 与回流说明

本文说明 Flutter Android MVP 当前已经成立的原生导航、外部来源、登录回流、公开链接、只读承接与轻互动边界。完整客户端范围仍以 `Clients/radish.flutter/README.md` 为准。

## 总体边界

Flutter 承担移动原生客户端路线，不复刻 WebOS。当前重点是公开内容阅读、原生返回、登录续接、个人复访、通知只读列表与 forum 回流、forum 评论 / 轻回应写入、公开商品只读浏览、登录态商城订单 / 背包只读查看、胡萝卜资产与经验记录只读查看、公开详情链接复制和 docs 原生阅读链路。

当前不包含：

- 完整通知中心
- 系统通知栏推送
- 完整商城工作台
- 购买、取消订单、支付口令、权益激活、道具使用
- 转账、打赏、调账、经验调整、冻结治理、管理员复核
- 完整资产中心、完整浏览历史治理
- 发帖编辑器、富文本评论、评论审核治理、点赞、投票、编辑治理
- 系统分享 SDK、海报生成、分享统计
- 桌面工作台多窗口能力

## 榜单到公开主页

原生 `榜单` tab 当前承载经验榜只读入口。

- 经验榜条目可打开原生公开主页。
- Android Back 返回榜单，而不是退到默认 profile tab 或退出应用。
- 商品榜、我的经验明细、经验交易记录和管理员调整能力不进入当前 Flutter 批次。

## 公开主页来源返回

原生公开主页是 Flutter 当前个人链路的复访承接点。

- 从发现页、forum 作者入口或榜单打开公开主页时，壳层会记录打开前 tab。
- Android Back 优先返回原来源 tab，而不是强制停留在 profile tab。
- 公开主页继续打开帖子、评论或轻回应详情后，详情页返回会回到公开主页；再次 Android Back 仍应回到最初来源 tab。
- 最近公开主页入口可保留最近一次公开用户，但该本地复访状态不等于完整浏览历史中心。

## 通知只读列表与 Forum 回流

已登录壳层会读取当前用户最近站内通知，作为移动端只读通知列表。

- 列表展示通知标题、内容、类型、已读状态和时间。
- forum 通知点击后复用 forum detail handoff 打开目标帖子。
- 通知携带 `commentId` 时，详情页继续尝试定位目标评论上下文。
- 系统通知等不可跳通知保持只读展示，不伪造目标详情。
- 从通知进入详情后，Android Back 返回打开通知前所在 tab，不强制留在 forum tab。
- 该入口不代表完整通知中心、系统通知栏推送、标记已读、删除通知或通知设置已纳入 Android MVP。

## Forum 评论发布与回复

Forum detail 当前在公开阅读基础上开放已登录态的根评论发布与评论回复。

- 匿名态可读取帖子正文、轻回应与评论区；从评论区发起登录后，OIDC 回跳会回到当前帖子评论输入上下文。
- 已登录态可发布根评论，也可回复根评论或子评论。
- 提交成功只局部更新评论区：根评论前插到评论列表，回复更新对应根评论的回复数和子评论列表。
- 提交失败只在评论输入区提示，不刷新正文、轻回应或来源 tab。
- 从公开主页评论上下文进入详情并回复后，Android Back 仍返回原公开主页来源。
- 发帖编辑器、点赞、投票、审核治理、富文本评论和完整社区治理仍不进入当前 Flutter 边界。

## Forum 轻回应登录回流

Forum detail 当前按“正文 -> 轻回应 -> 评论区”展示。

- 匿名态可读取最近轻回应。
- 已登录态可发布一句轻回应。
- 从轻回应区点击“登录后发布”后，浏览器 OIDC 登录成功会回到当前帖子轻回应区，并提示可继续发布。
- 发布成功只刷新轻回应墙与局部成功提示，不刷新正文或评论阅读位置。
- 轻回应删除、举报、审核台能力、点赞、投票和编辑治理仍不进入当前 Flutter 边界。

## Docs 原生阅读与内链

Docs detail 当前承载公开文档的原生只读阅读。

- `discover` 文档精选、docs 列表、docs 搜索结果和 profile 最近文档都可进入原生 docs detail，并按来源返回。
- 公开文档正文中的 `/docs/:slug`、完整公开 URL、`docs/:slug`、`./:slug` 与普通相对 slug 文档链接会继续打开原生 docs detail。
- 页内锚点、附件路径、外部链接和非 docs 链接仍按文本展示，不在 Flutter 内扩外部浏览器跳转或附件治理。
- 16 位以上纯数字旧 long slug 不作为公开可见 slug 展示或复制，避免把兼容路径当作普通用户可读地址。

## Shop 公开只读浏览

Flutter 当前只承接公开商城的移动只读浏览。

- 发现页可进入原生公开商城列表。
- 发现页商城精选商品和商城列表项可打开原生公开商品详情。
- Android Back 返回打开前来源：发现页来源回发现页，列表来源回商城列表。
- 商品详情只展示公开信息和只读购买边界；购买、支付口令和权益激活仍不进入 Flutter 当前边界。

## Shop 登录态订单与背包只读

已登录我的页当前承接商城私域的移动只读复访入口。

- 我的页可打开订单列表、订单详情和背包。
- 订单列表和订单详情读取当前用户订单、订单状态、商品快照和金额等信息。
- 背包展示当前用户权益和道具，并可从来源订单或来源商品继续打开只读详情。
- 从订单 / 背包进入来源页后，Android Back 返回原订单或背包上下文。
- Flutter 不承担购买、取消订单、支付口令校验、权益激活或道具使用；这些能力仍由 WebOS 私域商城和后续专门治理链路承接。

## 胡萝卜资产只读

已登录我的页当前提供胡萝卜资产只读入口。

- 资产页读取可用余额、冻结余额、累计获得、累计支出和最近流水。
- 最近流水用于移动端快速核对，不承担财务对账后台职责。
- Flutter 不开放转账、打赏、调账、支付或冻结治理。

## 经验记录只读

已登录我的页当前提供经验记录只读入口。

- 经验页读取等级、当前经验、总经验、升级进度、冻结状态和最近经验流水。
- 最近流水用于用户自查经验来源，不承担管理员复核或配置治理职责。
- Flutter 不开放经验调整、冻结治理或管理员复核。

## 公开详情链接复制

Flutter 原生 forum detail、docs detail 与 shop detail 使用与 Web 公开路由一致的复制口径。

- 复制内容由当前 `RADISH_GATEWAY_BASE_URL` 派生的 Gateway Base URL 加 Web 公开路径组成。
- forum detail 只有在 `Post.PublicId` 可用时展示可复制的 `/forum/post/:publicId`；旧 long `postId` 仍可作为打开兼容 fallback，但不作为复制口径。
- docs detail 使用 `/docs/:slug`；16 位以上纯数字旧 long slug 不生成公开复制链接。
- shop detail 使用 `/shop/product/:productId`。
- 来源返回、目标评论、当前 tab 等应用内上下文不写入复制链接。
- 当前只提供“展示完整公开链接 + 复制到剪贴板”，不接系统分享 SDK、海报生成或分享统计。

## 标识与内部接口

- 外部来源优先携带 `postPublicId`。
- 旧 `postId` 只作为字符串 fallback。
- 进入详情并读取成功后，评论、轻回应和定位类内部接口使用真实 `VoId`。
- 不应把公开标识误传给需要内部 ID 的接口。

## 验证要点

- 经验榜用户条目打开公开主页后，Android Back 回榜单。
- 从 discover 或 forum 作者入口打开公开主页后，Android Back 回原来源 tab。
- 从公开主页打开帖子 / 评论详情并返回后，再按 Android Back 仍回到公开主页原来源 tab。
- 从非 forum tab 打开通知列表并选择 forum 通知进入详情后，返回回到原 tab；系统等不可跳通知保持只读展示。
- 通知携带 `commentId` 时，应落到评论上下文或给出可理解的定位失败提示。
- 匿名态从评论区发起登录后，应回到当前帖子评论输入上下文；已登录态发布根评论或回复后，只局部更新评论区。
- 匿名态从轻回应区发起登录后，应回到当前轻回应区并可继续发布。
- 发现页进入公开商城列表和商品详情后，Android Back 应回到原来源。
- 我的页进入订单列表、订单详情、背包、来源订单或来源商品后，Android Back 应回到原订单 / 背包 / profile 上下文。
- 我的页进入胡萝卜资产和经验记录后，应能读取概要、分页或最近流水、空态 / 错误态，并可返回 profile。
- forum / docs / shop detail 展示的公开链接应能完整复制，长链接在窄屏下不撑开布局。
- docs 正文内链命中公开 docs 路径或相对 slug 时，应继续打开原生 docs detail；附件、锚点和外部链接不应误当作原生文档跳转。
