# 页面真实联调与浏览器 Smoke 规则

> 本页说明 Radish 在真实页面联调、浏览器 smoke 和人工页面复核时的视图覆盖口径。
>
> **最后更新**: 2026.06.10

## 适用范围

当任务需要打开真实页面验证用户可见行为时，默认适用本规则，包括：

- 纯 Web 公开页面、登录态页面和 `/desktop` 保留入口。
- Console 管理后台页面。
- 登录回流、来源返回、深链、静态资源、Gateway 转发和页面级交互。
- 修复用户截图中反馈的布局、状态、跳转、实时刷新或可见文案问题。

只改后端纯算法、数据库查询、接口契约、文档或测试，且不需要打开页面确认用户可见结果时，可以不执行浏览器视图联调；但最终回复里应说明本轮没有页面 smoke。

## 默认入口

浏览器真实联调默认优先访问 Gateway：

```text
https://localhost:5000
https://localhost:5000/console/
```

`http://localhost:3000` 与 `http://localhost:3100` 仅用于 HMR、前端资源路径或局部 UI 调试。涉及登录、回流、公开壳层、Console base、API 转发或部署口径时，不应默认绕过 Gateway。

## 视图覆盖

进行页面真实联调或浏览器 smoke 时，默认同时覆盖 PC 与移动端视图。

推荐基线：

- PC：默认使用 `1920x1080` 桌面视口；如需覆盖更高分辨率，可补 `2560x1440`。不再把 `1280x720` 作为完整 PC smoke 基线，只能作为窄桌面兼容补充。
- 移动端：默认使用真实高分屏移动设备口径，即移动 CSS 视口配合高 DPR，而不是把 CSS viewport 直接写成物理像素。推荐 `390x844 @ DPR 3`（物理约 `1170x2532`）或 `412x915 @ DPR 3`（物理约 `1236x2745`）。
- 若工具无法设置 DPR，应在记录中明确说明；此时 `390x844` / `412x915` 只能代表移动布局宽度，不能代表 1080p 以上物理屏 smoke。

补充视图：

- 窄桌面、低分辨率或折叠屏等边界视图只在任务明确影响响应式布局时追加，不替代默认 PC / 移动基线。
- Console 或表格类页面如需要移动端复核，也优先使用上述高分屏移动口径；若该页面不承诺移动主形态，应记录限制和剩余风险。

检查重点：

- 首屏是否能正常渲染，关键内容不空白、不被遮挡。
- 主导航、返回、tab、按钮、输入框和列表滚动在两种视图下都可用。
- 文案、徽标、计数、空态、错误态和加载态不溢出、不重叠。
- 登录回流、来源返回、公开详情跳转、用户主页跳转等 URL 状态在刷新后仍稳定。
- Console 或表格类页面在移动视图下至少不阻断关键查看路径；若移动端不是主形态，应明确说明限制。

如果任务明显影响响应式布局、移动 Web、公开分享入口、浮层、输入区、固定工具条或滚动容器，应扩大移动端检查范围，不只看一个首屏。

## 记录口径

交付总结中应说明：

- 已验证的入口和账号状态。
- 已覆盖的 PC 与移动端视图。
- 关键路径是否通过。
- 因账号、数据、服务状态或工具限制未覆盖的部分。

若只完成 PC 或只完成移动端，应明确写出原因和剩余风险，不把单视图通过写成完整页面联调通过。

## 参考原则

本规则参考同体系项目的真实路径分层验证口径：用户可见页面变更应覆盖主要使用路径，并按风险补充失败态、刷新态和窄屏状态，而不是只验证单一桌面状态。
