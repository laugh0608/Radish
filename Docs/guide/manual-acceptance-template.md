# 人工验收模板

> 本页定义 Radish 当前推荐的“最小人工验收”结构，用于减少专题文档之间的口径漂移。
>
> 回归执行后的结果记录建议见：[回归结论记录模板](/guide/regression-result-template)

## 推荐结构

建议每个需要人工验收的专题至少补齐以下字段：

1. **适用改动范围**
   - 说明什么类型的改动后需要回来看这份验收模板
2. **前置条件**
   - 需要启动哪些宿主
   - 是否需要管理员 / 普通用户双账号
   - 是否需要测试素材、现成数据或浏览器多端环境
3. **联调资产**
   - 对应的 `HttpTest` 文件
   - 必要时附带脚本、前端测试或专题索引入口
4. **最小人工验收顺序**
   - 只保留最短主链，按执行顺序编号
   - 每一步只写“做什么 + 看什么结果”
5. **预期结果**
   - 把“通过标准”收束为几条稳定结论
6. **可跳过项**
   - 标明哪些只在 UI / 实时 / 上传 / 多端场景相关改动后才需要执行
7. **结论记录格式**
   - 给出最小记录模板，便于周志或 PR 复用

## 推荐写法

### 1. 适用改动范围

- 频道列表、消息发送、撤回、未读同步改动后
- 发帖、评论、投票、问答、抽奖详情交互改动后
- 文档树、导入导出、版本回滚改动后

### 2. 前置条件

- 已启动 `Radish.Api`、`Radish.Auth`、`Radish.Gateway`
- 已通过 `Radish.Api.AuthFlow.http` 获取 `access_token`
- 已准备管理员账号 / 普通用户账号 / 测试素材

### 3. 联调资产

- `Radish.Api.Tests/HttpTest/Radish.Api.Chat.http`
- `Radish.Api.Tests/HttpTest/Radish.Api.Wiki.http`
- [专题回归索引](/guide/regression-index)

### 4. 最小人工验收顺序

1. 打开入口并确认基础列表 / 详情可加载。
2. 执行最短写操作主链，确认保存或提交成功。
3. 刷新列表 / 详情，确认状态、统计或展示结果已回写。
4. 补一个失败或边界场景，确认不是静默失败或误返回 500。

### 5. 预期结果

- 主链路可完成，且刷新后状态一致。
- 关键边界场景有明确反馈，不出现静默回退。
- 若涉及多端 / 实时，另一端在合理时间内能看到一致结果。

### 6. 可跳过项

- 仅样式调整时，可跳过接口层 `HttpTest`
- 仅后端 DTO / Service 改动且未影响实时或 UI 时，可跳过窗口层观察

### 7. 结论记录格式

```md
- 验收日期：2026-03-17
- 验收人：<name>
- 验收范围：<topic>
- 执行入口：<http / ui / script>
- 结果：通过 / 阻塞
- 备注：<如有失败点，记录复现条件与影响范围>
```
