## 当前关注（2026-03-14）

### 正在进行

1. **P0 权限冻结后的短回归观察期**
   - `Console` 权限治理 V1 已切换为“冻结边界、只做回归维护”
   - 权限链路变更优先通过 `npm run check:console-permissions` 自检
   - 仅在真实新增页面/接口时增量补矩阵、文档与种子
   - 不再继续横向扩张新的权限族或共享接口映射

2. **聊天室 P1 收口确认**
   - `@mention`、图片消息、引用回复、断线补拉、成员头像来源、乐观发送与失败重试已完成
   - 最新一轮已修复长整型 ID 精度问题与输入区布局挤压问题，并完成一轮手工冒烟联调
   - 当前无明显阻塞问题，已具备从短观察切向下一候选主线的条件

3. **身份治理尾项收尾**
   - 外部兼容确认后推进 Claim 协议输出收敛
   - 防回归扫描规则留待 `M13` 纳入脚本 / CI

4. **工程治理**
   - `DbMigrate` 已完成本轮解耦宿主、启动入口收口与 `doctor` 只读校验命令补齐
   - 持续减少“组合根直接绑宿主”的结构耦合

5. **P2 开源软件清单声明组件**
   - 欢迎 App 内的最小首版已落地，提供项目级核心依赖与许可证说明入口
   - 当前先采用静态清单方案，避免把中小体量补齐项膨胀为构建治理任务
   - 仅当发布或合规场景提出明确需要时，再推进到完整 `THIRD-PARTY-NOTICES` / 自动生成能力

6. **P3 论坛投票 MVP 已完成验收**
   - 已确认采用“挂在论坛帖子中”的实现路径，不单独拆独立投票 App
   - 数据模型 / DTO / Vo、发帖链路、投票接口、前端详情交互、列表轻量摘要与验收回归已全部收口
   - 截止时间临近当前时刻的 UTC 口径问题已修复，手工联调已通过
   - 后续如继续推进，转入 `P3-ext`，不再占用当前主线

7. **P4 论坛问答 MVP 已完成一轮联调收口**
   - 已按“论坛帖子附带问答”的路径完成问答帖发布、回答提交、提问者采纳与详情交互补齐
   - 回答输入已复用现有 Markdown 编辑器，列表与详情均已补齐问答状态展示
   - 后续如继续推进，转入 `P4-ext`，聚焦问答视图、排序与历史能力

### 暂不作为当前主线

- 治理前端化：审核台、敏感词与自动策略
- 分发增强：更复杂召回 / 排序 / 观测压测
- 国风 UI / 主题切换
- i18n 语言切换与键值治理
- 邮件通知系统
