## 当前关注（2026-03-18）

### 当前阶段原则

1. **先主线，后细节**
   - 当前优先完成真正影响首个 dev 版本可用性的主功能、主链路与验证入口，不再持续深挖社区局部体验增强。

2. **先闭环，后打磨**
   - 只要功能已形成稳定闭环、没有明显断链或结构性脆弱点，就先进入总回归与发布准备，而不是反复抠小交互。

3. **不留坑，不留技术债**
   - 允许实现保持克制，但不允许把明显缺口、长期已知风险、脆弱契约或结构性脏债留到首个 dev 版本之后再补。

4. **非阻塞增强统一后置**
   - `P3-ext`、`P4-ext`、`P5-ext`、`Console-ext Phase 2+` 等增强项统一后置到首个 dev 版本之后再评估，不再抢占当前主线。
