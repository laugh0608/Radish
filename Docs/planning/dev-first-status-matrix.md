# 首版 dev 功能矩阵状态表

> 本页用于把 [首版 dev 边界](/planning/dev-first-scope) 里的功能范围进一步落成 **状态矩阵**。
>
> 状态只允许使用四类：
>
> - `已完成`：主线闭环已完成，当前更偏向总回归确认。
> - `待联调复核`：主线大体具备，但仍需按首版 dev 视角做一轮统一联调或烟雾复核。
> - `待补齐`：仍存在首版范围内尚未完成的主线缺口，不能直接判定为已完成。
> - `不纳入首版`：明确后置，不参与首版 dev 是否达标的判断。
>
> 复核日期：`2026-03-25`

## 3. 当前结论

按当前矩阵，首版 `dev` 还没有达到“可发内部开发版”状态，原因不在于社区主线还没做完，而在于以下两类事项仍未完成：

1. **待联调复核项仍较多**
   - WebOS 桌面与应用容器
   - 论坛基础
   - 社区 P0
   - Console V1
   - 国风视觉基线
   - 验证基线
   - Docker 镜像构建链

2. **已完成项需要等待总回归统一确认**
   - 通知中心、投票、问答、抽奖、聊天、商城、文档、萝卜坑、经验等级等主线虽然已完成，但仍需在首版视角下一次性串联确认

## 4. 建议执行顺序

1. 继续维护 `radish.client` 的主题切换 / i18n，并对已完成首轮烟雾验证的桌面壳层、个人中心与欢迎 App 做更大范围联调复核。
2. 并行安排 WebOS、论坛基础、社区 P0、Console 的首版烟雾联调。
3. 以当前验证基线与最小 CI 门禁为底座，对已补齐的 Docker 镜像资产继续做 Compose 运行态 Smoke 与首版交付口径收口。
4. 最后基于本页状态，把“待联调复核”逐项转成“已完成”。
