# 第二阶段产品功能补全规划

> 状态：进行中
>
> 最后更新：2026-05-06（Asia/Shanghai）
>
> 本页承载第二阶段从“多端路线验证”回到“产品功能补全”后的详细优先级、端侧分工和近期批次建议。快速入口仍以 [当前进行中](/planning/current) 为准。

## 背景判断

第二阶段已经完成公开内容壳层、Flutter Android MVP 与 Tauri + WebOS 桌面安装包路线的首轮验证。当前不应继续按端侧验证或分发产品化材料作为默认主线，而应回到产品本身，补齐首版发布后遗留的真实功能坑、治理能力和体验断点。

当前文档中部分模块被标记为“已完成”，其含义主要是首版主链路完成；代码和专题文档仍保留了模拟实现、无动作入口、治理缺口和后置增强项。后续应重新按产品价值和风险排序，而不是把所有后置项永久搁置。

## 端侧任务占比

| 方向 | 占比 | 当前职责 |
| --- | ---: | --- |
| WebOS / PC 工作台 | 45% | 补齐已登录用户完整工作流，重点是萝卜坑、个人中心、通知、商城、论坛工作台交互与跨应用回流 |
| 后端 + Console 治理 | 25% | 补后台管理、统计、权限、安全、经验治理、商城管理与运维可见性缺口 |
| Flutter 移动端 | 15% | 保持高价值移动路径，不复刻 WebOS；优先最小已登录链路、个人复访、通知入口和轻操作 |
| 公开 Web 壳层 | 10% | 只处理真实阅读、分享、响应式、来源返回和公开访问质量问题 |
| Tauri 桌面壳 | 5% | 继续维护观察；签名、自动更新、菜单、托盘和公开分发能力仍在真实分发前再评估 |

## 优先级原则

1. 优先补“用户能点到但不能真正完成”的功能，例如模拟 API、空操作按钮、`alert` 占位和跳转 TODO。
2. 优先补会影响资产、安全、权益、经验和治理可信度的能力。
3. WebOS 工作台承载“做事情”的完整流程，公开壳层承载“看内容”的低门槛访问，Flutter 承载移动高频路径。
4. 不因为某能力在首版矩阵里写过“已完成”，就忽略当前代码中的真实缺口。
5. 不默认扩完整 PWA、完整 E2E、完整可观测性平台、开放平台或新 BFF。

## 近期批次建议

### P2-C1 萝卜坑工作流补全

目标：让萝卜坑从“可展示”推进到更可靠的日常资产工作台。

- 接通支付密码设置 / 修改真实 API，移除模拟 `setTimeout` 和 `alert`
- 快捷操作真正切换到转移、记录、安全、统计
- 最近交易“查看全部”切到交易记录
- 复制流水号改为统一 toast 反馈
- 安全日志若后端已有能力则接入；若没有，明确回写为后端治理缺口

### P2-C2 Console 与后台治理补洞

目标：把已经存在的后端能力和 Console 页面补成可用管理面。

- 用户详情、萝卜币流水、订单列表不再停留在模拟 API
- 设置页与个人资料页的保存动作改为真实能力或明确后置
- 商城管理前端从 M10 后置项回拉评估：商品、订单、统计优先级分层
- `StatisticsController` 中的统计 TODO 判断保留、实现或废弃

### P2-C3 经验 / 等级治理

目标：补齐社区长期运营需要的经验治理能力。

- 管理员调整、冻结、解冻从框架推进到真实逻辑
- 每日统计更新和等级配置缓存补齐
- 每日上限从硬编码转为配置读取
- 经验值防刷和异常检测先做最小可解释规则，不做大而全风控平台

### P2-C4 商城权益效果收口

目标：避免用户可购买但权益效果不完整。

- 复核双倍经验、帖子置顶、帖子高亮对应商品是否对用户可见 / 可买
- 若可买，优先接通真实效果或从商品种子 / 展示中收回
- 订单超时取消、权益过期检查按真实风险排序，不直接做完整定时任务平台

### P2-C5 移动端高价值已登录链路

目标：让 Flutter 移动端继续补真实使用价值，而不是转向分发材料。

- 评估最小通知入口与 forum detail 回流
- 评估个人复访中的轻操作补强
- 不复刻桌面工作台，不扩完整通知中心、发帖、完整评论提交、点赞、投票或编辑治理

## 当前不做

- Android / Tauri 的正式分发产品化材料优先级不高于产品功能补全
- 不继续围绕 WebOS 复访入口做低收益微调
- 不把所有公开壳层页面继续扩细节
- 不启动 Flutter iOS 产品化工程
- 不启动完整 PWA / 离线能力 / Web Push
- 不启动完整 Playwright / E2E 平台
- 不启动完整可观测性平台或开放平台

## 文档同步规则

- `current.md` 只保留当前主线、占比和下一顺位摘要。
- `development-plan.md` 保留路线图总览和端侧任务占比。
- `backlog.md` 只保留明确后置项；被回拉的事项应迁出 backlog 或在本页列入近期批次。
- 功能实现细节、命令级验证记录和批次流水写入开发日志或专题文档，不堆到入口页。
