## 当前目标

1. **启动 P3-8-A 多端功能缺口与 UI 设计入口审计**
   - 建立覆盖公开页、移动 Web 公开视图、Flutter、WebOS / PC / Tauri、Console 的功能缺口矩阵
   - 区分未开发、未适配、只读缺口、登录态缺口、跨端回流缺口、壳层能力缺口和治理效率缺口
   - 梳理哪些端点需要 Pencil 设计稿先行，哪些小范围功能 / 文案 / 状态修正可以直接实现
   - 从矩阵中选出 `1-2` 个一天级、可验证、能明显推进产品的首批开发任务
2. **维护观察降级为并行线**
   - `P3-6 / P3-7` 暴露的新问题仍可回拉小闭环，但不再占用主线
   - 维护线只处理公开访问、head / sitemap、分享预览、运行日志、购买 / 订单 / 背包、权限授权等高信号问题
3. **保留关键门禁**
   - 页面级 UI 重设计、客户端级视觉治理和 Console 大页面收敛原则上先更新 Pencil 设计稿
   - 纯功能缺口、后端治理、行为等价拆分、小范围表单 / 状态 / 文案修正不被 Pencil 前置阻塞
   - 不把完整 E2E、完整可观测性平台或大而全运营平台作为继续开发前置
