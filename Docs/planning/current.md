## 当前目标

1. **推进 P3-8-D 多端主路径缺口补齐与验收矩阵**
   - 先建立移动 Web 公开视图验收矩阵，覆盖 `/discover / forum / docs / u/:id / leaderboard / shop` 的窄屏信息密度、来源返回、分享入口和公开只读边界
   - 根路径 `/` 已先切向 `/discover` 纯 Web 公开分发页，后续继续评估纯 Web 主路径补强或 Flutter 下一批高价值功能，不默认继续做 Console 剩余页面微调
   - WebOS 只维护 `/desktop` 保留入口和迁移过渡，不作为新增功能目标
   - 只回拉能影响真实访问、移动阅读、跨端回流、购买 / 订单 / 背包、权限授权或治理效率的高信号问题
2. **维护观察降级为并行线**
   - `P3-6 / P3-7` 暴露的新问题仍可回拉小闭环，但不再占用主线
   - 维护线只处理公开访问、head / sitemap、分享预览、运行日志、购买 / 订单 / 背包、权限授权等高信号问题
3. **保留关键门禁**
   - 大页面重设计、端点级视觉治理、跨页面视觉体系变更原则上先更新 Pencil 设计稿
   - 纯功能缺口、后端治理、行为等价拆分、小范围表单 / 状态 / 文案修正不被 Pencil 前置阻塞
   - 不把完整 E2E、完整可观测性平台或大而全运营平台作为继续开发前置
