## 2. 阻断修复方案

### 2.1 Discover 内容优先重构

- 删除首屏摘要指标、路线说明矩阵和正常数据态下四个分域完整列表。
- 主体固定为一条社区帖子 / 活动流，将热门标签与参与入口作为辅助内容。
- Docs、商城和榜单只保留与社区活动相关的少量辅助卡片，其余入口由“更多”或对应域页面承接。
- 先同步 `Docs/frontend/public-web-unified-experience-design.md` 与 Pencil `P02` 对照画板，再修改代码；加载、错误和空态继续保留可行动反馈。

### 2.2 关系复访契约

- 在公开个人页接入既有 `GetFollowStatus`、`Follow`、`Unfollow`，不新增后端关系接口。
- 登录态展示明确的“关注 / 已关注 / 取消关注 / 处理中”状态；自己的公开主页不展示关注动作。
- 匿名点击关注时使用受控回流 `/u/:id?intent=follow`，登录回调后恢复到同一公开主页并继续呈现待完成的关注动作，不自动替用户提交关注。
- `authReturnPath` 只接受合法用户标识与唯一 `intent=follow`，拒绝额外参数、重复参数、hash、非法 LongId / PublicId，避免把普通公开 URL 全量放入登录回流白名单。
- 补公开路由、登录回流、静态契约和关注服务定向测试；圈子关注列表 / 动态和关注通知沿用现有后端能力。

### 2.3 公开 Docs 服务端契约

- 在现有 `Wiki` 控制器下新增专用匿名读取动作：`PublicGetList`、`PublicGetTree`、`PublicGetBySlug`。
- 服务层新增对应公开读取方法，查询条件固定为未删除、`Status=Published`、`Visibility=Public`，不读取当前登录态、角色或权限；公开分页总数、树和详情都以同一服务端谓词为准。
- 既有 `GetList`、`GetTree`、`GetBySlug` 继续服务作者态 / 登录态读取，`Admin*` 继续服务 Console，不改变现有调用语义。
- `publicDocsApi.ts` 切换到 `Public*` 动作并显式 `withAuth: false`；删除把前端过滤当安全边界的实现。展示层可以保留契约断言，但不能借此放宽服务端结果。
- 补控制器与服务定向测试，覆盖草稿、`Authenticated`、`Restricted`、软删除、分页总数、树节点和 slug 详情均不可进入公开结果。

## 3. 成组实施顺序
