## 分级清单

### 必须先修复的阻断项

截至 E6，未发现需要在进入 `P3-12-F` 前必须先修复的接口、后端行为、错误模型、权限、审计或运行时契约缺口。

### 发布前建议项

这些项目不阻断进入 `P3-12-F`，但应在 F 内完成验证、接受记录或发布边界说明：

- 刷新发布候选验证矩阵：`validate:baseline`、`validate:identity`、`validate:baseline:host` 或对应阶段入口、`git diff --check` 和仓库卫生检查。
- 在用户当日确认前后端已启动后，执行 Gateway PC / mobile 发布候选 smoke，覆盖公开冷启动、登录后复访、论坛、Docs、商城、通知 / 消息、Console 治理和错误恢复代表页。
- 对 E5-B 未执行的路径给出 F 级处理：注册提交、聊天发送、轻回应 / 问答回答、最终购买扣费、Console 高风险治理动作、真实登录过期、断网 / 弱网和慢接口。
- 复核运营默认值：角色权限、分类 / 标签、默认频道、商品、经验、宠物、通知规则和系统设置在候选环境中的可交付状态。
- 补发布候选可访问性抽样：键盘可达、焦点可见、表单错误、触控尺寸、移动输入和长文本，优先覆盖发帖、评论、聊天、结算和 Console 审核。
- 对 `/legal` 和用户承诺材料做发布前文案校对，明确虚拟商品、不退款边界、社区规则、隐私说明和未成年人 / 敏感内容处理口径仍不是正式律师审阅文本。

### 可后置项

这些项目需要保留产品边界和替代路径，但不阻断进入 `P3-12-F`：

- 完整私聊、公开聊天室发现、聊天搜索、Reaction 和完整移动聊天回复流。
- 通知偏好、归档、静音、多端同步和批量处理。
- 跨域全局搜索、推荐系统和内容分发算法。
- 完整钱包、退款 / 售后、资产风控、完整支付争议处理和经济系统扩展。
- 屏蔽 / 拉黑、申诉工单、自动化治理、敏感词平台和举报滥用评分。
- 独立移动 Console、内部 Jobs 平台、远端错误监控、服务端诊断编号和支持工单系统。
- 账号导出 / 注销、完整数据留存自动化说明和正式法务协议平台。
- Flutter 原生复访、系统通知和 WebOS 保留入口的非默认路径清理。
