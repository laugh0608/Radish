# P3-12-D67 正式产品成熟度专题立项记录

> 日期：2026-07-05（Asia/Shanghai）
> 范围：`P3-12-D` UI 专题退出判断修正、`P3-12-E` 新专题立项、发布候选顺延

## 背景

`P3-12-D66` 已完成 host runtime 与 Gateway public / private / console PC、mobile 成组运行态补验，未发现新的阻断级真实页面问题。

随后用户明确指出：正式产品不能只以“没有新的阻断”为标准，还需要检查功能完整、安全、产品力、UI / 人机交互友好、可靠性等更高维度，并建议增加新的大专题来处理。同时用户补充观察：Console 看起来尚未形成明确移动端视图。

## 决策

新增 `P3-12-E 正式产品成熟度与质量硬化`，并将原计划的正式版发布候选专题顺延为 `P3-12-F`。

此决策意味着：

- `P3-12-D` 可结束为 UI 与当前发布前页面族首批实现 / 补验阶段，但不直接进入发布候选。
- `P3-12-E` 承接正式产品成熟度检查和补强，不把“无阻断 smoke”当作发布质量结论。
- Console 移动视图进入 E 期高优先审计项：区分“已有响应式基础”“正式移动治理视图不足”和“独立移动 Console 后置”三种结论。

## 新专题范围

- 功能完整度：公开、私域、作者、交易、Docs、Console 真实任务。
- 安全与权限：公开只读边界、Console 权限、写入幂等、高风险动作、敏感日志。
- 产品力：入口发现、任务连续性、成功 / 失败 / 空态反馈、登录恢复。
- UI / 人机交互：PC / mobile 信息层级、表单、抽屉、弹窗、表格、触控密度。
- 可靠性：慢接口、大列表、刷新恢复、Hub 恢复、上传、Markdown、错误态。
- 发布就绪：必修复清单、后置清单、PR / 发布候选验证入口。

## 首批顺位

1. `P3-12-E1 正式产品成熟度矩阵与差距审计`。
2. `P3-12-E2 Console 移动治理视图与高风险后台交互硬化`。
3. `P3-12-E3` 至 `E6` 再按 E1 矩阵排序。

## 保持不变

- 不创建 tag。
- 不进入 M15 测试或生产部署。
- 不把独立移动 Console App、内部 Jobs 平台、新治理 API 直接塞进当前批次。
- 若 E 期确认必须新增 API、权限键、数据库或保存载荷，必须先补专题方案和验证口径。
