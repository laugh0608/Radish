## 纠偏结论

`P3-12-D49` 的正确结论是：当前已实现页面在 Gateway PC / mobile 真实页面复核中未暴露阻断级问题。

它不能推出以下结论：

- 不能说明整个 app 已经按 Pencil 设计稿完成 UI 实现。
- 不能替代 D37 中标记为“首轮落地待观察”的移动任务流与复杂响应式页面继续对齐。
- 不能把 D38 已裁决后置的公开聊天室、内部调度中心、内部 Jobs 平台和独立移动 Console 写成已完成或当前缺口。
- 不能直接进入 `P3-12-E` 发布候选准备。

因此，`P3-12-D` 继续作为当前主线。下一批应继续做 UI 实现，而不是整理发布候选材料。

## 下一批排序

### 第一批：`P3-12-D51 radish.client 私域 / 作者态移动任务流 UI 对齐首批`

优先原因：

- 对应 `private-web-workflows.pen` 的 `P21-P30`，在 D37 中仍有多项“首轮落地待观察”。
- 覆盖正式用户高频路径：个人状态、内容、资产、订单、通知、消息、圈子、宠物、论坛作者态和 Docs 作者态。
- 当前页面与 API 均存在，主要是 UI 对齐、移动密度、状态槽、卡片层级、底栏空间和任务节奏问题，不需要先新增数据模型。

建议覆盖：

- `/me`、`/me/content`、`/me/history`、`/me/attachments`、`/me/experience`
- `/me/assets`、`/me/assets/transactions`、`/shop/orders`、`/shop/inventory`
- `/notifications`、`/messages`、`/circle`、`/pet`
- `/forum/compose`、`/docs/mine`

本批目标：

- 对齐移动端信息密度、摘要卡、状态槽、筛选区和列表 / 卡片节奏。
- 控制页面级横向溢出，保留必要的内部滚动。
- 不改变 API、权限、路由、保存动作、提交载荷或业务状态机。

### 第二批：`P3-12-D52 Public Web 移动公开任务流 UI 对齐`

- 覆盖 `/discover`、`/forum`、论坛详情、`/docs`、Docs 详情、`/shop`、商品详情、`/leaderboard` 和公开主页。
- 重点复核公开移动首屏、详情页阅读节奏、公开参与入口和低频入口 `/workbench` 的承接关系。

### 第三批：`P3-12-D53 Console 移动响应式后台 UI 精修`

- 覆盖内容治理、经验治理、订单 / 商品 / 胡萝卜、文档治理、角色权限和系统设置。
- 重点处理 Console 移动端的页面标题、指标、筛选、表格滚动、抽屉 / 弹窗和权限反馈。
- 不把 Console 改成独立移动应用。

## 本批不做

- 不进入 `P3-12-E`。
- 不创建 tag、不恢复 PR、不进入 M15 测试 / 生产部署流程。
- 不新增公开聊天室、内部调度中心、内部 Jobs 或独立移动 Console。
- 不新增业务 API、权限键、数据库结构或保存载荷。
- 不修改 Pencil 设计源；下一批代码实现时按现有设计源和说明文档对齐。
