# P3-12-E5-A 用户可恢复错误与反馈硬化第三批记录

> 日期：2026-07-06（Asia/Shanghai）
>
> 范围：`Frontend/radish.client`、`Docs/planning`。本批不改后端接口、HTTP 错误模型、错误上报链路、日志链路、数据库结构或跨端运行时行为。

## 目标

承接 `E5-A` 前两批的发帖、评论、Workbench、聊天失败消息和通知目标缺失反馈，继续补齐页面级错误恢复与 Hub 状态说明：

- 私域资产页、公开 Docs、商城订单 / 详情 / 背包在加载失败时提供重试与可复制诊断。
- 通知和聊天 Hub 状态说明从“当前状态”补到“恢复后怎么处理”，让用户知道页面数据、草稿或失败消息是否仍可继续使用。
- 复制诊断只包含定位元信息，不复制聊天正文、文档搜索关键词、订单详情、资产流水内容或通知正文。

## 修改摘要

### 统一诊断复制

- 新增 `recoveryDiagnostics` 前端工具，统一生成页面级恢复诊断文本。
- 诊断内容包含模块、阶段、当前路径、错误信息、时间戳和必要目标元信息。
- 诊断工具复用现有剪贴板能力，不新增网络上报、服务端诊断编号或 Console 工单。

### 资产与 Docs

- `/me/assets` 资产概览 / 流水加载失败时，在保留刷新动作的基础上增加“复制诊断”。
- 复制内容包含资产页面模式、分页、筛选状态和是否已有余额数据，不包含资产流水明细。
- 公开 Docs 列表、搜索和详情错误态增加“复制诊断”。
- Docs 搜索诊断只记录是否存在关键词，不复制关键词本身；详情诊断记录 slug 和锚点状态。

### 商城订单与背包

- 商城数据 hook 增加 `loadError`，区分订单列表、订单详情和背包加载失败。
- `/shop/orders`、`/shop/order/:id` 和 `/shop/inventory` 在加载失败时提供重试和复制诊断。
- 诊断内容包含路由类型、页码、订单 ID 或背包加载阶段等定位信息，不复制订单明细、商品明细或背包内容。

### Hub 状态恢复说明

- 聊天频道连接异常横幅增加恢复说明：草稿和本地失败消息保留，连接恢复后可重试，频道未追平时可手动刷新。
- 通知正式 Web 页侧栏增加连接状态说明，区分在线同步、恢复中和离线状态。
- 本批只补用户可见说明，不改 SignalR 连接策略、重连参数、未读计算或消息 / 通知状态机。

## 不纳入本批

- 不新增服务端诊断编号、错误上报 API、Console 支持单、日志关联查询或审计查询入口。
- 不做弱网模拟平台、断网注入、接口错误注入或跨端 E2E 平台。
- 不改变商城订单、资产流水、Docs 公开阅读、聊天 Hub 或通知 Hub 的接口契约。
- 不把完整钱包、退款 / 售后、资产风控、完整聊天平台或通知偏好设置混入本批。
