## 2. 已确认链路

1. 商品详情可稳定加载长 ID 商品，不再因前端数值精度或路由参数转换失败导致详情异常。
2. 订单详情可查看商品关联信息，并能从订单跳转到对应商品或用户治理上下文。
3. 从订单跳转到用户 / 商品后，返回链路可回到原订单上下文，不丢失治理追查路径。
4. 商品删除前会展示明确确认，不再允许误触后静默删除。
5. 已存在任意订单记录的商品，后端删除动作会被拒绝，避免破坏历史订单追溯。
6. 商品相关订单入口可用于删除前人工判断，前后端删除口径保持一致。

## 3. 剩余处置

- 商城管理本轮不再继续追加低收益微调，转入稳定维护。
- 后端 + Console 治理下一顺位切回剩余历史构建 warning 与少量安全治理尾项。
- 若后续改动再次触达商品删除、订单详情、长 ID 参数或跨页返回链路，应复跑本记录覆盖的最小人工链路。
