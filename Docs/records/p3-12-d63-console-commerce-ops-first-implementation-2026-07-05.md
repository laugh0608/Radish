# P3-12-D63 Console 商业运营页面族首批实现记录

> 日期：2026-07-05（Asia/Shanghai）
> 范围：`radish.console` 商品管理与订单管理
> 设计源：`Docs/frontend/design-sources/console-governance-workbench.pen`

## 背景

`P3-12-D63 Console` 已完成内容治理 / 经验治理工作台首批实现。按 `Docs/planning/current.md` 的下一顺位，本批继续对照 Console 设计源处理商业运营页面族。

本批通过 Pencil MCP 只读抽取 `console-governance-workbench.pen` 的 `P10 - Console Commerce Operations`。设计源强调：商品和订单不是孤立表格，应把商品池、上架状态、库存售卖、订单失败、发放幂等和最近订单留痕放到同一套运营任务面里；移动 Console 继续只作为响应式后台任务流参考，不实现独立移动端 Console。

## 实现范围

### 商品运营

- `/console/products` 在既有筛选、表格、详情、编辑、上下架和相关订单动作上补商品运营任务流。
- 任务流按商品池、上架状态、库存售卖和订单回流展示当前页运营状态。
- 右侧摘要升级为商业运营摘要，展示库存预警、当前商品、类型 / 库存、价格 / 销量和暂不支持购买原因。
- 当前商品动作继续复用现有详情、相关订单、编辑和上架 / 下架处理函数，不新增跳转语义或提交字段。
- 库存统计兼容无限库存商品，不把无限库存误判为库存预警。

### 订单运营

- `/console/orders` 在既有 URL 筛选、表格、详情、用户 / 商品 / 资产流水跳转和发放重试动作上补订单运营任务流。
- 任务流按订单范围、失败优先、交易金额和留痕回流组织管理员扫描顺序。
- 右侧摘要展示待处理订单、失败订单、完成订单、交易金额和当前订单上下文。
- 当前订单动作继续复用现有详情、查看用户、查看商品、资产流水和失败发放重试逻辑。
- 订单状态判断兼容后端枚举值、字符串状态和展示文本，避免数字枚举导致失败 / 完成 / 待处理归类偏移。

## 保持不变

- 不新增 API、权限键、数据库结构、路由语义或保存 / 提交载荷。
- 不改变商品分类、商品分页、详情抽屉、编辑抽屉、上下架、商品表单字段、商品上传或相关订单筛选逻辑。
- 不改变订单筛选、详情抽屉、发放重试、用户详情、商品详情、资产流水回流或权限判断。
- 不实现完整资产风控、退款 / 售后、支付口令、独立支付审计、发货工单或新的商业治理 API。
- 不启动 Gateway PC / mobile 真实页面 smoke；本轮未由用户明确确认前后端已启动。
